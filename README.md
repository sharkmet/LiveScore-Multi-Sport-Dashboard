# LiveScore Dashboard

A real-time sports dashboard covering MLB, NHL, NBA, and NFL. Live game data is polled from official/public APIs and pushed to a React frontend over WebSocket. Each sport has a fully custom game detail view and a statistical win probability model.

---

## Features

### Live Scores
- All 4 leagues — MLB, NHL, NBA, NFL — on a single scoreboard
- WebSocket push: score, clock, and game state update without page refresh
- Day selector: Yesterday / Today / Tomorrow with live indicator
- League filter: All / MLB / NHL / NBA / NFL
- Infinite-scroll ticker at the top with live game cards (wraps to fill the strip)

### Game Detail Views

**MLB**
- Inning-by-inning linescore (R / H / E)
- Play-by-play timeline with pitch/hit events
- Win probability chart (Recharts area chart, per-play)
- Team stats panel (hits, errors, LOB, etc.)

**NHL**
- P1 / P2 / P3 / OT / SO linescore always shown regardless of current period
- Play-by-play feed: goals highlighted, penalties/shots/hits/blocks shown — sticky top-3 most-recent plays, older plays scrollable below
- 6-stat team comparison bars: SOG, Power Play, PIM, Hits, Blocked Shots, Faceoff %
- Win probability chart with intermission detection (WP paused during breaks)

**NBA**
- Quarter-by-quarter linescore
- Scoring plays feed with sticky top-3 most-recent plays
- Game leaders panel (points, rebounds, assists) height-matched to the WP chart
- Win probability chart with quarter-break and halftime labels

**NFL**
- Situation bar: down & distance, field position, game clock
- Drive timeline with drive result labels
- Timeout indicators (home and away)
- Red zone indicator
- Win probability chart with halftime detection

### Win Probability Models

Each sport uses a purpose-built model — not a generic logistic regression:

| Sport | Model | Notes |
|-------|-------|-------|
| MLB | API cascade → Tango win expectancy table → logistic fallback | 9,072-state lookup table generated by Markov chain simulation; prefers real `liveData.plays` WP when available |
| NHL | Poisson-probit | PP boost (+0.18 λ), 5-on-3 multiplier (1.8×), empty-net adjustments, OT/SO special cases |
| NBA | Stern probit | `P = Φ(lead / (13.5 × √t))` with home-court advantage (2.7 pts) and team ratings |
| NFL | 15-feature logistic regression | Score diff, time remaining, timeouts, down & distance, yard line, momentum; hard rules in final minutes |

Charts display evenly spaced period/quarter tick labels regardless of when the server was started, with a live "end tag" showing the current probability.

### Alert Engine
- **Upset** — underdog (≥5 pp lower win%) leads by 3+ in inning/period 5+
- **Comeback** — team that trailed by 4+ now leads
- **Blowout** — run/goal differential reaches 10+
- **No-hitter watch** — one team has 0 hits through inning 6+
- **Momentum shift** — 3+ runs scored in latest half-inning, margin ≤ 1
- Each alert fires at most once per game per server session
- Toast notifications slide in from the corner; auto-dismiss after 6 s

### Standings & Playoffs
- Division standings for all four leagues (pulled from live APIs)
- Playoff bracket view (NHL, NBA, NFL playoff rounds)

### UX
- Dark / light theme toggle (dark default)
- Loading skeletons while WebSocket delivers the first snapshot
- Error boundary with "Try again" button
- Mock data fallback — if the backend is unreachable after 5 s, mock games load automatically

## Tech Stack

| Layer | Tech |
|-------|------|
| Backend | Node.js 18+, Express, `ws` |
| Frontend | React 18, Vite, Tailwind CSS, Recharts |
| Transport | WebSocket (full-duplex push) |
| Data — MLB | MLB Stats API (`statsapi.mlb.com`) |
| Data — NHL | NHL API (`api-web.nhle.com`) |
| Data — NBA | ESPN APIs |
| Data — NFL | ESPN APIs |

## Getting Started

### Prerequisites

- Node.js 18 or later (uses built-in `fetch` — no `node-fetch` needed)

### Backend

```bash
cd server
npm install
node src/server.js
```

Listens on `http://localhost:3001` by default.

### Frontend

```bash
cd client
npm install
npm run dev
```

Open `http://localhost:5173`.

---

## Environment Variables

| Variable | Default | Purpose |
|---|---|---|
| `PORT` | `3001` | Backend server port |
| `USE_MOCK_DATA` | `false` | Force mock data for all leagues |

```bash
PORT=3002 USE_MOCK_DATA=true node src/server.js
```
